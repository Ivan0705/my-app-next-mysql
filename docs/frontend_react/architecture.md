Критерии Feature:
1. Реализует конкретную бизнес-фичу - конвертация SQL файлов

2. Имеет собственный UI + логику - компонент + состояния

3. Может быть использован на разных страницах - может быть переиспользован

4. Изолирован от других фич - независимый модуль

## Почему FSD (Feature-Sliced Design)?

### Проблемы, которые решает FSD:
1. **Запутанные зависимости** - невозможно понять, что от чего зависит
2. **Сложность рефакторинга** - изменение одного компонента ломает другие
3. **Медленный онбординг** - новым разработчикам сложно разобраться
4. **Низкая переиспользуемость** - компоненты сильно связаны между собой
